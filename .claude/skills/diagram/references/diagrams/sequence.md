# 时序图规范（Sequence Diagram）

> 双轨制：日常用 D2 快速出图，正式文档用 HTML/SVG 精确控制。

---

## 适用场景

- API 调用链路（登录、支付、数据同步）
- 微服务间消息交互
- 客户端-服务端通信流程

---

## 模式选择

| 场景 | 模式 | 工具 |
|------|------|------|
| 日常画图、快速出图 | 快速模式 | D2（`shape: sequence_diagram`） |
| 调研报告、方案文档 | 精品模式 | HTML/SVG |

---

## 组件使用规则

| 元素 | 快速模式（D2） | 精品模式（HTML/SVG） |
|------|-------------|-------------------|
| 参与者 | 圆角矩形，每个独立配色 | 同左 |
| 数据库 | `shape: cylinder` | SVG 圆柱 |
| 生命线 | 虚线（D2 自动） | 1px dashed #CBD5E1 |
| 激活条 | ❌ 不支持 | 10px 宽，#DBEAFE 填充，#93C5FD 边框 |
| 同步消息 | `->` 实线箭头 | 实线 + 实心三角箭头 |
| 返回消息 | `->` 实线（D2 不区分） | 虚线 + 开放箭头 |
| 自调用 | `A -> A` | 右侧绕回弧线 |

---

## 参与者配色

按调用链路从左到右排列，每个参与者使用主题色序列：

| 位置 | 典型角色 | 色系 |
|------|---------|------|
| 1 | 用户/客户端 | C-1 Blue |
| 2 | 前端/BFF | C-2 Emerald |
| 3 | 网关/代理 | C-3 Amber |
| 4 | 业务服务 | C-5 Violet |
| 5 | 数据库 | N（灰色系） |
| 6 | 缓存/MQ | C-4 Rose |

---

## 布局规则

1. **流向**：固定纵向（参与者在上，时间向下）
2. **参与者间距**：80-120px，均匀分布
3. **消息间距**：纵向 24-32px
4. **消息标签**：在箭头上方，带序号（1. 2. 3.）
5. **参与者数量**：不超过 6 个
6. **消息数量**：不超过 15 条

---

## 快速模式（D2）

```d2
shape: sequence_diagram

参与者A: {
  style: { fill: "#EFF6FF"; stroke: "#93C5FD"; font-color: "#1E293B"; border-radius: 6; font-size: 13 }
}
数据库: {
  shape: cylinder
  style: { fill: "#F8FAFC"; stroke: "#CBD5E1"; font-color: "#1E293B"; font-size: 13 }
}

参与者A -> 参与者B: 消息内容
```

生成命令：`d2 --theme 0 --pad 40 input.d2 output.png`

---

## 模板

- 快速模式：`templates/d2/sequence.d2`
- 精品模式：`templates/html/sequence.html`

---

## 禁忌

- 参与者不超过 6 个
- 消息不超过 15 条
- 不使用设计规范以外的颜色
