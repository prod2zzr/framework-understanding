# 架构图规范（Architecture Diagram）

> 仅 HTML/CSS 模式。D2/Mermaid 的图论布局无法精确控制分层堆叠。

---

## 适用场景

- 微服务系统架构
- 技术栈分层
- 模块依赖概览

---

## 设计原则

架构图展示的是**层级关系**（哪些组件在哪一层），不是调用关系（调用链路用时序图）。因此：
- **不画连线**：层与层之间用虚线分隔 + 协议标注代替箭头
- **分层堆叠**：从上到下按层级排列
- **同层等宽**：同层组件用色块平铺，等高等间距

---

## 布局结构

```
┌──────────────────────────────────────────┐
│  标题 + 副标题                             │
├──────────────────────────────────────────┤
│  L-1  │ 层标签     │ [组件] [组件] [组件]   │
│       │            │                       │
├ - - - - - 协议/接口标注 - - - - - - - - - ┤
│  L-2  │ 层标签     │ [组件] [组件] [组件]   │
│       │            │                       │
├ - - - - - 协议/接口标注 - - - - - - - - - ┤
│  L-3  │ 层标签     │ [组件] [组件]          │
└──────────────────────────────────────────┘
```

---

## 层级配色

使用设计规范的层级色（L-1 ~ L-6），从上到下：

| 典型层级 | 层级色 | 组件色块 |
|---------|--------|---------|
| 客户端 / 接入层 | L-1 (Blue) | `#3B82F6` 填充 + 白字 |
| 业务服务层 | L-2 (Emerald) | `#10B981` 填充 + 白字 |
| 中间件 | L-3 (Amber) | `#F59E0B` 填充 + 白字 |
| 数据层 | L-4 (Violet) | `#8B5CF6` 填充 + 白字 |
| 基础设施 | L-5 (Rose) | `#F43F5E` 填充 + 白字 |
| 外部系统 | L-6 (Cyan) | `#06B6D4` 填充 + 白字 |

---

## 组件样式

| 属性 | 值 |
|------|-----|
| 形状 | 圆角矩形 |
| 圆角 | `4px` |
| 填充 | 对应层级色主色 |
| 文字 | `12px Medium` 白色 |
| 间距 | 组件间 `8px` |
| 高度 | `32px` 统一 |
| padding | `0 14px` |

## 层分隔

| 属性 | 值 |
|------|-----|
| 分隔线 | `1px dashed #E2E8F0` |
| 协议标注 | 居中，`10px Regular`，颜色 `#94A3B8` |
| 示例 | `HTTPS / WebSocket`、`gRPC / REST` |

---

## 整体尺寸

- 宽度：`860-960px`
- 背景：白色，`border-radius: 12px`，微阴影
- 内边距：`28-32px`
- 层高：自适应（根据组件行数）

---

## 层数限制

- 不超过 7 层
- 每层组件不超过 8 个
- 超出时按业务域拆分为多张架构图

---

## 模板

`templates/html/architecture.html`

---

## 生成方式

1. 生成 HTML 文件
2. Playwright 截图为 PNG

---

## 禁忌

- 不画具体连线（用时序图/数据流图展示调用关系）
- 不使用渐变色
- 不使用 icon/emoji
- 不使用设计规范以外的颜色
