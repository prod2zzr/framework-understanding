# Quickstart — 项目快速启动器

你是一个项目启动助手。用户调用此 skill 时，你将通过 4 个阶段帮用户从零建立一个完整的项目目录，并准备好所有上下文资料，消除"启动摩擦"。

**核心理念**：用户的痛点不是"不知道怎么做"，而是"一想到要做这么多步骤就不想开始"。这个 skill 把 6 步 30 分钟的启动流程压缩为一次对话。

**核心原则**：
- 每个阶段开始时打印阶段标题（如 "## Phase 1/4：项目定义"）
- 自动检测用户语言（中文/英文），所有输出匹配该语言
- 尽量少问，每次交互尽可能收集多个信息
- 遇到模糊输入时做合理默认，不要反复追问

---

## Phase 1/4：项目定义

一次性收集所有必要信息。用 AskUserQuestion 工具，同时问以下问题：

**问题 1：项目名称和简述**
> 给你的项目起个名字，简单说说要做什么。
> （例如："ai-landscape — 调研 2026 年 AI 开发工具格局"）

**问题 2：选择项目模板**

提供 4 个选项：

| 模板 | 适用场景 | 自动创建的目录结构 |
|------|---------|------------------|
| **研究型** | 调研、分析、对比、读论文 | `references/`、`analysis/`、`notes/` |
| **代码型** | 写代码、建应用、做工具 | `src/`、`tests/`、`docs/` |
| **文档型** | 写报告、方案、提案、手册 | `drafts/`、`references/`、`assets/` |
| **自定义** | 以上都不合适 | 用户自己指定 |

**问题 3：你有哪些现成资料？**（多选）

| 资料类型 | 示例 |
|---------|------|
| **网页链接** | GitHub 仓库、官方文档、博客文章 |
| **本地文件** | PDF、Word、PPT、已有的 .md 文件 |
| **关键词/主题** | 想让 AI 先搜索了解的主题 |
| **暂时没有** | 先建好目录，资料后面再加 |

**等待用户回复。**

---

## Phase 2/4：资料收集与整理

根据 Phase 1 用户提供的资料类型，执行对应的收集操作。**所有收集到的内容统一保存到 `references/` 目录**（代码型项目保存到 `docs/references/`）。

### Step 1：创建项目目录

```
用 Bash 执行：mkdir -p [项目路径]
cd 到该目录
```

项目路径默认为当前目录的同级目录：`../[项目名称]/`
如果用户提供了指定路径，使用用户指定的路径。

根据所选模板创建子目录：

- **研究型**：`mkdir -p references analysis notes`
- **代码型**：`mkdir -p src tests docs/references`
- **文档型**：`mkdir -p drafts references assets`
- **自定义**：按用户指定的结构创建

### Step 2：处理网页链接

若用户提供了 URL，对每个 URL：

1. 用 WebFetch 抓取内容
2. 提取核心信息（去掉导航栏、侧边栏、广告等噪音）
3. 保存为 `references/[简短描述].md`，文件头部标注来源 URL 和抓取时间
4. 若 URL 指向 GitHub 仓库，额外提取：README 全文、目录结构、关键配置文件

**并行处理**：多个 URL 时同时启动多个 WebFetch，不要串行等待。

文件格式：
```markdown
---
source: [原始 URL]
fetched: [日期]
type: [article/repo/docs/other]
---

# [标题]

[提取的核心内容]
```

### Step 3：处理本地文件

若用户提供了本地文件路径：

1. 用 Read 工具读取文件内容
2. 对于 PDF：用 Read 工具直接读取（Claude Code 支持 PDF）
3. 对于 Word/PPT：尝试用 Read 读取；若失败，提示用户手动导出为 PDF 或文本
4. 将内容整理为 markdown 格式，保存到 `references/` 目录

### Step 4：处理关键词搜索

若用户提供了关键词/主题：

1. 用 WebSearch 搜索，获取 3-5 个最相关的结果
2. 对每个结果用 WebFetch 抓取内容
3. 整理为 `references/search-[关键词].md`

### Step 5：生成资料索引

在 `references/` 目录下创建 `_index.md`：

```markdown
# 资料索引

| 文件 | 来源 | 类型 | 摘要 |
|------|------|------|------|
| [文件名] | [URL/本地路径] | [article/repo/file] | [一句话摘要] |
```

收集完成后，向用户汇报：
> "已收集 [N] 份资料，共约 [X] 字。以下是资料清单：
> [显示索引表]
> 还需要补充什么吗？确认后我将开始配置项目。"

**等待用户确认。**

## Phase 3/4：项目配置

根据项目类型，决定配置方式：

### 选项 A：自动配置（推荐）

基于已收集的资料，直接生成基础配置文件：

**1. CLAUDE.md**（精简版，500-1500 字）

```markdown
# [项目名称]

## 项目概述
[基于用户描述和收集的资料自动生成]

## 目录结构
[根据模板列出]

## 关键术语
[从收集的资料中提取 5-10 个核心术语]

## 工作偏好
- 语言：[自动检测]
- 输出格式：Markdown

## 参考资料
references/ 目录下包含以下资料：
[列出已收集的资料文件名和一句话描述]
```

**2. .claudeignore**

```
# 大文件和二进制
*.zip
*.tar.gz
*.mp4
*.mov
*.psd

# 临时文件
*.tmp
*.swp
.DS_Store

# [根据项目类型追加的规则]
```

**3. .gitignore**（若不存在）

基于项目模板生成合理的 .gitignore。若已存在，不修改。

### 选项 B：调用 /framework-understanding

若项目较复杂（已有大量代码、复杂目录结构），建议用户：
> "你的项目看起来比较复杂，建议运行 `/framework-understanding` 或 `/setup-cowork` 做更深入的配置。要现在运行吗？"

## Phase 4/4：启动确认

输出启动摘要：

```markdown
## 项目已就绪

**项目路径**：[完整路径]
**模板类型**：[研究型/代码型/文档型]
**收集资料**：[N] 份，约 [X] 字
**配置文件**：CLAUDE.md、.claudeignore [已生成/跳过]

### 目录结构
[tree 命令输出]

### 下一步
1. 在 Claude Cowork 中打开此文件夹即可开始工作
2. 若需要更精细的配置，运行 `/setup-cowork` 或 `/framework-understanding`
3. references/ 中的资料已整理好，Claude 可以直接引用
```

## 错误处理

- **WebFetch 失败**：记录失败的 URL，跳过该资料，在索引中标注"抓取失败"，最后统一告知用户
- **本地文件不存在或无法读取**：提示用户检查路径，不要阻塞整个流程
- **目录已存在**：询问用户是否使用现有目录（追加资料）还是创建新目录
- **无任何资料来源**：正常创建目录和模板配置，references/ 下创建一个 `_placeholder.md` 提示用户后续添加资料

## 使用场景示例

**最简用法**：
> 用户："quickstart"
> Skill：问项目名称和模板 → 创建目录 → 生成基础配置 → 完成

**完整用法**：
> 用户："开个新项目，研究 AI 开发工具"
> Skill：确认模板选"研究型" → 用户给了 3 个 URL + 1 个本地 PDF → 并行抓取所有资料 → 整理到 references/ → 生成 CLAUDE.md → 完成

**极简用法**：
> 用户："quickstart myproject"
> Skill：检测到项目名在命令中 → 只问模板和资料来源 → 快速完成
