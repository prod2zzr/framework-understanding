# 知识库（Knowledge Base）

本目录存放 HR 领域知识文件，供 LLM 在分析花名册时自动加载使用。

## 设计原则

- **文件格式**：`.md`（Markdown），人可直接编辑，LLM 可直接理解
- **与代码解耦**：知识文件独立于 Python 脚本，修改知识不需要改代码
- **按主题组织**：每个 `.md` 文件对应一个知识主题
- **结果另存**：分析结果存 `.json`，知识规则存 `.md`，职责分明

## 文件命名规范

```
<主题>.md
```

示例：
- `职称评审条件.md` — 各级别职称的认定/评审标准
- `学历认定规则.md` — 学历等级、同等学力的认定方法
- `岗位匹配标准.md` — 不同岗位的任职资格要求
- `年限计算规则.md` — 工龄、任职年限的计算方法

## 编写规范

每个知识文件应包含：
1. **标题**：说明本文件涵盖的知识主题
2. **适用场景**：说明 LLM 在什么情况下应该使用这份知识
3. **具体规则**：用清晰的条件-结论格式描述规则
4. **注意事项**：容易混淆或特殊处理的情况

## 使用方式

LLM 在分析花名册时，会通过 `load_knowledge` 工具按主题加载相关知识文件。
HR 无需理解具体的筛选条件，系统会自动应用这些规则。
但 HR 应在结果中抽样检查（sample check），确认筛选结果的准确性。
